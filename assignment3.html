def readFood(folder):
    """
    Input: Name of the folder to read the excel files from
    Output: A dataframe with information for each fruit (one per file). Each 
    fruit is a row and there are seven columns.
    """
    # Read each file and save in fileContent:
    files = [x for x in os.listdir(folder) if not x.startswith("~")] # Get all file names
    os.chdir(folder) # Enter the folder with those files
    fileContent = [pd.read_excel(fname, sep = '\t') for fname in files] # Read each file and save in a list
    
    os.chdir("..") # Move back out to the directory we started in for future reads
    
    # Pull out the fresh row from each dataframe and modify it to the format we want
    count = 0
    for fc in fileContent:
        
        # Eliminate unnecessary columns that some files have and rename columns:
        if fc.shape[1] > 7:
            for j in xrange(7, fc.shape[1]):
                fc = fc.drop('Unnamed: ' + str(j), 1)
        fc.columns = ['form', 'price_per_lb', 'type', 'yield', 'lb_per_cup', 'food', 'price_per_cup']
                
        # Some files have many types of that produce, so we want to catch those:
        problem = False
        if fc.loc[2, "price_per_lb"] == " ":
            problem = True
        elif math.isnan(fc.loc[2, "price_per_lb"]):
            problem = True
                
        # Insert appropriate values for type and food:
        fc.loc[2, "type"] = folder
        fc.loc[2, "food"] = files[count].split(".")[0]
            
        # Drop all but the second row, drop the second row if not Fresh, and drop problem produce
        fc = fc.drop( xrange(3, fc.shape[0]))
        if fc.loc[2, "form"].find("Fresh") == -1:
            fc = fc.drop([2])
        elif problem == True:
            fc = fc.drop([2])
        fc = fc.drop([0, 1])
        fileContent[count] = fc
        count += 1
    
    # Combine into one DataFrame: 
    together = pd.concat(fileContent)    
    
    return together

To look for something interesting about the data, I was curious about the relationship between yield and price per pound. It is my understanding that yield is the amount usable after the produce is processed (peeled, cooked, ...). So it would make sense that you would pay more for a higher yield. Looking at the fruits and vegetables together, there is a positive linear relationship. This goes along with my hypothesis. There are a few possible outliers, but overall what I was expecting. Comparing this relationship for fruits and vegetables, the fruit relationship appears to have a higher slope. So the fruits have higher prices with higher yields than the vegetables. This is probably due to the fact that vegetables are cooked while fruits are usually not. Cooking probably reduces the yield since a lot of vegetables tend to shrink when cooked.
